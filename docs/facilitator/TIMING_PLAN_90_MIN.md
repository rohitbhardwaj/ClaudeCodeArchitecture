# 90-Minute Timing Plan

## Recommended Flow

| Segment | Time |
|---|---:|
| Opening and framing | 5 min |
| Act 1 — From Vibe Coding to Governed Engineering | 10 min |
| Act 2 — What Claude Code Really Changes | 10 min |
| Act 3 — Governed Repo Context | 10 min |
| Act 4 — Governed Workflow | 10 min |
| Act 5 — Governed Katas | 10 min |
| Act 6 — Enterprise Controls | 10 min |
| Act 7 — Adoption and Learning Loop | 10 min |
| Act 8 — Production Scenarios | 10 min |
| Act 9 — Closing | 5 min |

## Minimal Assignment Set

For a tight 90-minute delivery, use only these:

```text
Act 5: Review the AI-Generated Diff Kata
Act 6: Safe / Ask / Never Policy Design
Act 7: Claude Code Adoption Metrics
Act 8: Architecture Review Board Simulation
```

## If Time Is Short

Skip detailed worksheet completion and instead ask participants to answer:

```text
Approve, request changes, or block?
Why?
What evidence is missing?
What control would prevent this next time?
```

## If Time Is Long

Expand into a half-day workshop by running:

```text
repo context mapping
CLAUDE.md rewrite
control plane map
rollout readiness score
production change readiness review
architecture review board simulation
```
