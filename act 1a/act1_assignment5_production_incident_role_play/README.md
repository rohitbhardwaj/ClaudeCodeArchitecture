# Act 1 Assignment 5 — Production Incident Role Play

## Theme

The failure was not:

```text
AI wrote code.
```

The failure was:

```text
AI-generated change entered the system without architecture controls.
```

## Scenario

A Claude-generated PR was merged.

Two days later:

```text
Checkout error rate increased.
Some valid coupons are rejected.
Customer support tickets increased.
Logs contain partial customer email addresses.
Tests are still passing.
```

## Roles

Split participants into four roles:

1. Developer
2. Architect
3. Security reviewer
4. Product owner

## Group Output

Each group creates a 5-part incident review:

```text
1. What changed?
2. What should have been caught before merge?
3. Which prompt constraint was missing?
4. Which quality gate was missing?
5. What rule should be added to CLAUDE.md?
```

## What Is Included

This artifact kit includes:

- Assignment instructions
- Incident timeline
- Simulated PR summary
- Evidence packet
- Role cards
- Group worksheet
- Incident review template
- Facilitator guide
- Debrief script

- CLAUDE.md rule recommendations
- Better restart prompt
- Quality gate checklist

## Recommended Duration

35–45 minutes for workshop mode.

15–20 minutes for conference mode.
