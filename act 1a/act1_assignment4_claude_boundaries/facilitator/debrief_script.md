# Debrief Script

Use this after the worksheet and discussion.

## Opening

Most teams start with the wrong question:

```text
Can Claude Code do this?
```

The better question is:

```text
Should Claude Code do this without approval?
```

## Key Message

Claude Code should not have the same authority for every task.

Explaining code is not the same as changing authentication.

Generating tests is not the same as deleting tests.

Drafting docs is not the same as modifying public APIs.

Suggesting refactors is not the same as rewriting large modules.

## Teaching Point

Governance is not a blocker.

Governance is how teams make acceleration repeatable and safe.

## Closing Line

The first production rule for Claude Code is simple:

```text
Define the boundary before you delegate the work.
```
