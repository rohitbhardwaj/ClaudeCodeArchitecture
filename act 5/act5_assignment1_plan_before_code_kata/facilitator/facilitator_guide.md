# Facilitator Guide — Plan Before Code Kata

## Learning Objective

Participants practice converting a vague implementation-first prompt into a plan-first governed prompt.

## Setup Script

Say:

```text
The weak request is: Fix the checkout bug.
This is how vibe coding starts.
Your job is to rewrite it so Claude analyzes before editing.
```

## 3-Minute Live Version

| Segment | Time |
|---|---:|
| Show weak prompt | 30 sec |
| Participants rewrite | 90 sec |
| Show strong answer | 60 sec |

## 8-Minute Workshop Version

| Segment | Time |
|---|---:|
| Explain risk | 1 min |
| Inspect worksheet | 1 min |
| Draft prompt | 3 min |
| Pair compare | 2 min |
| Debrief | 1 min |

## Facilitation Prompts

Ask:

```text
Does your prompt stop Claude from editing?
What context should Claude inspect?
What behavior must remain unchanged?
What tests should Claude identify?
Where does the human checkpoint happen?
```

## Strong Closing Line

The first Claude Code prompt should often be analysis, not implementation.
