# Scoring Rubric — Plan Before Code Kata

## 5 — Excellent

Prompt is analysis-first and includes all key governance elements:

```text
do not edit yet
inspect repo context
identify root cause
list behavior to preserve
small implementation plan
tests needed
risks
approval checkpoint
```

## 4 — Strong

Prompt is analysis-first but misses one element.

## 3 — Acceptable

Prompt asks for some analysis but permits coding too early.

## 2 — Weak

Prompt asks Claude to fix and test with minimal governance.

## 1 — Poor

Prompt is implementation-first.
