# Quick Kata Scorecard

Score the diff review.

| Criterion | Points |
|---|---:|
| Identifies controller boundary issue | 1 |
| Identifies public API error-code change | 1 |
| Identifies PII logging risk | 1 |
| Identifies removed failing test | 1 |
| Identifies happy-path-only test gap | 1 |
| Identifies missing rollback | 1 |
| Identifies missing approvals | 1 |
| Blocks the diff | 1 |
| Provides safer response to Claude | 1 |

## Score

```text
/9
```

## Interpretation

```text
8–9: Strong governance review
6–7: Good review but missed one risk
4–5: Partial code review, incomplete governance
0–3: Unsafe approval risk
```
