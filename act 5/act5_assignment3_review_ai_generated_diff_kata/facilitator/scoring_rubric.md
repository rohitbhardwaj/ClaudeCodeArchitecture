# Scoring Rubric — Review the AI-Generated Diff Kata

## 5 — Excellent

Participant blocks the diff and identifies all major risks:

```text
architecture boundary
API error-code change
PII logging
removed test
happy-path-only coverage
missing rollback
missing approvals
```

## 4 — Strong

Participant blocks the diff and identifies most major risks.

## 3 — Acceptable

Participant requests changes but misses some governance risks.

## 2 — Weak

Participant only catches code-level issues.

## 1 — Poor

Participant approves the diff.
