# Debrief Script

## Opening

The score is not about judging the team.

It is about understanding whether the repository gives Claude Code enough structure, guidance, and guardrails.

## Key Message

Claude Code will amplify repository maturity.

If the repo has:

```text
clear structure
tests
contracts
CLAUDE.md
security rules
quality gates
ownership
rollback patterns
```

then Claude has better context.

If the repo has:

```text
tribal knowledge
weak tests
stale docs
unclear boundaries
no review ownership
```

then Claude has more room to infer incorrectly.

## Important Point

Tool rollout should follow repository readiness.

The question is not:

```text
Can we turn Claude Code on?
```

The question is:

```text
What repository maturity do we need before scaling usage?
```

## Closing Line

Claude Code adoption is not only a tooling decision.

It is a repository maturity decision.
