# One-Page Handout — Agent Blast Radius Exercise

## Change Request

```text
Add support for promotional discount codes during checkout.
```

## Blast Radius Template

```text
1. User-visible behavior:
2. API contract:
3. Pricing:
4. Tax:
5. Payment:
6. Refunds:
7. Fraud/abuse:
8. Data model:
9. Analytics:
10. Support:
11. Observability:
12. Tests:
13. Rollback:
14. Required approvals:
```

## Strong Answer Highlights

```text
Pricing changes payment amount.
Tax basis may change.
Refunds must account for discounts.
Fraud controls are needed.
Support needs rejection reasons.
Observability must be PII-safe.
Feature flag rollback is required.
Approvals include architecture, product, security, finance, and QA.
```

## Key Lesson

```text
Before Claude changes code, architects must understand the blast radius.
```
