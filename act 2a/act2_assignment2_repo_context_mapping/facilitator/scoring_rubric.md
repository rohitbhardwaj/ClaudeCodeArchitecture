# Scoring Rubric — Repo Context Map

## 5 — Excellent

Participant identifies:

```text
entry point
business logic
external dependencies
data persistence
tests
API contract
security-sensitive areas
inspect-only files
approval-required files
forbidden files
```

Answer shows understanding of blast radius.

## 4 — Strong

Participant identifies most categories but may miss one such as API contract or security-sensitive logging.

## 3 — Acceptable

Participant identifies main production files but misses tests, API contract, or forbidden files.

## 2 — Weak

Participant only lists the file where the bug appears.

## 1 — Poor

Participant moves directly to code changes without context mapping.
