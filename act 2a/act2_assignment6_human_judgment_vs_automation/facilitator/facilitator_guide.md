# Facilitator Guide — Human Judgment vs Claude Automation

## Learning Objective

Participants learn to separate what Claude Code can automate from what humans must decide.

## Setup Script

Say:

```text
Claude Code can accelerate many parts of engineering.
But speed does not transfer accountability.
In this exercise, we separate automation from judgment.
```

## Timing

### Conference Version

| Segment | Time |
|---|---:|
| Explain categories | 2 min |
| Individual classification | 6 min |
| Pair compare | 3 min |
| Debrief | 4 min |

Total: 15 minutes

### Workshop Version

| Segment | Time |
|---|---:|
| Explain decision rights | 5 min |
| Individual classification | 8 min |
| Group debate | 10 min |
| Reflection | 5 min |
| Debrief | 5 min |

Total: 33 minutes

## Facilitation Prompts

Ask:

```text
Which responsibility was hardest to classify?
Which human decision are teams tempted to delegate?
Where can Claude safely accelerate work?
Where must human accountability remain explicit?
What decision are your tools influencing too much today?
```

## Common Mistake

Participants may classify “identify security risks” as Claude can automate.

Clarify:

```text
Claude can help identify security risks, but security validation remains shared with human experts.
```

Participants may classify “draft ADR” as Claude can automate.

Clarify:

```text
Claude can draft an ADR, but humans own the decision.
```

## Strong Closing Line

Claude can recommend. Humans remain accountable.
