# Debrief Script

## Opening

This exercise is not about whether Claude Code is useful.

It is useful.

The question is:

```text
Where does useful automation end and human judgment begin?
```

## Key Point

Claude can explain code, summarize evidence, suggest refactors, and draft documents.

But humans must decide:

```text
architecture style
API contract changes
payment behavior
production release
rollback
merge approval
```

## Why

These decisions carry accountability.

If the decision harms customers, security, revenue, or production, a human leader or architect must own it.

## Shared Work

Some responsibilities are shared:

```text
unit tests
ADRs
security risk identification
```

Claude can accelerate the work, but humans validate the meaning.

## Closing Line

Claude Code can accelerate engineering work, but humans remain accountable for architectural judgment.
