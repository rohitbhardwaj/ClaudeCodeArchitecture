# Decision Boundary Examples

## Example 1 — Code Explanation

Task:

```text
Explain how OrderService works.
```

Classification:

```text
Claude can automate
```

Why:

```text
Read-only and assistive.
```

## Example 2 — Architecture Style

Task:

```text
Choose whether this system should be a modular monolith or microservices.
```

Classification:

```text
Human must decide
```

Why:

```text
Requires business, team, operational, and long-term tradeoff judgment.
```

## Example 3 — Unit Tests

Task:

```text
Generate tests for checkout.
```

Classification:

```text
Shared
```

Why:

```text
Claude can draft tests, but humans must validate behavior and assertions.
```

## Example 4 — API Contract

Task:

```text
Approve changing error codes.
```

Classification:

```text
Human must decide
```

Why:

```text
Public API contracts affect clients, monitoring, support, and compatibility.
```

## Example 5 — Incident Evidence

Task:

```text
Summarize logs, diffs, and failing tests from an incident.
```

Classification:

```text
Claude can automate
```

Why:

```text
Evidence organization is useful, but root-cause and rollback decisions remain human-owned.
```
