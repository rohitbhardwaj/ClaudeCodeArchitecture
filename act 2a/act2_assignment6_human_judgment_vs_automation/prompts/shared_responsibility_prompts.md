# Prompts for Shared Responsibilities

## Generate Unit Tests

```text
Draft unit tests for this behavior.

Do not modify production code.

For each test, explain:
1. Behavior protected.
2. Failure path covered.
3. Why the assertion is meaningful.
4. Whether this is new behavior or existing behavior.

Do not remove or weaken existing tests.
```

## Draft ADR

```text
Draft an ADR for this architecture decision.

Mark it as a draft.

Include:
1. Context.
2. Options considered.
3. Decision proposal.
4. Tradeoffs.
5. Risks.
6. Consequences.
7. Open questions.
8. Human approvals required.
```

## Identify Security Risks

```text
Identify potential security risks in this change.

Do not claim the review is complete.

Include:
1. Input validation risks.
2. Authorization risks.
3. Secrets or PII risks.
4. Logging risks.
5. Dependency risks.
6. Recommended human security review areas.
```
