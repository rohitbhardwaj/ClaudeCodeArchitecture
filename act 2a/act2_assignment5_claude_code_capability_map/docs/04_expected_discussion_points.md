# Expected Discussion Points

Participants should discover that Claude Code is useful beyond coding.

## High-Value, Low-Risk Areas

Often good early adoption areas:

```text
Discovery
Documentation drafts
PR summaries
Test suggestions
Legacy code explanation
Risk analysis
```

## Higher-Risk Areas

Require stronger controls:

```text
Coding in production modules
Modernization rewrites
Incident fixes
Payment/pricing/auth changes
Database migrations
Public API changes
```

## Key Discussion Question

```text
Which delivery stage gives your team the highest value with the lowest risk?
```

## Possible Strong Answers

### Discovery

High value because Claude can explain unfamiliar code and map dependencies without changing files.

### Testing

High value if tests are reviewed and used to improve coverage.

### Documentation

High value if docs are reviewed against code.

### Review

High value if Claude summaries are evidence-based and do not replace human review.

## Caution

Coding may feel like the obvious answer, but it is often not the safest first adoption stage.
