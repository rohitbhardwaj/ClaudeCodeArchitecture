# Facilitator Guide — Claude Code Capability Map

## Learning Objective

Participants learn to place Claude Code intentionally across the delivery lifecycle instead of using it randomly during coding.

## Setup Script

Say:

```text
Most teams ask: How can Claude write code faster?
Architects should ask: Where does Claude fit safely into our delivery lifecycle?
```

## Timing

### Conference Version

| Segment | Time |
|---|---:|
| Explain lifecycle map | 2 min |
| Individual map | 6 min |
| Pair compare | 4 min |
| Group debrief | 4 min |

Total: 16 minutes

### Workshop Version

| Segment | Time |
|---|---:|
| Explain lifecycle stages | 5 min |
| Individual map | 10 min |
| Group map | 10 min |
| Pilot selection | 6 min |
| Debrief | 5 min |

Total: 36 minutes

## Facilitation Prompts

Ask:

```text
Where is Claude useful without changing code?
Which stage has the highest value with lowest risk?
Which stage has the highest risk?
What guardrail would make coding safer?
What guardrail would make incident usage safer?
```

## Common Mistake

Participants may focus only on coding.

Redirect:

```text
Claude Code may create safer early value in discovery, testing, review, and documentation.
```

## Strong Closing Line

Do not randomly introduce Claude Code where developers happen to use it first.

Place it intentionally where the lifecycle needs acceleration and where controls are ready.
