# Scoring Rubric — Capability Map

## 5 — Excellent

Participant maps all lifecycle stages and clearly connects:

```text
capability
risk
guardrail
human accountability
```

Identifies a practical low-risk pilot stage.

## 4 — Strong

Participant maps most stages and identifies useful risks and guardrails.

## 3 — Acceptable

Participant maps stages but guardrails are generic.

## 2 — Weak

Participant focuses mainly on coding and misses lifecycle-level thinking.

## 1 — Poor

Participant treats Claude Code as a generic tool without risks or guardrails.
