# Coding Prompts

## Focused Change

```text
Implement the approved change only.

Approved scope:
[insert scope]

Constraints:
- Change only approved files.
- Preserve public API behavior.
- Do not change payment, pricing, tax, or auth behavior without approval.
- Do not remove or weaken tests.
- Keep the PR small.

After coding, summarize changed files and tests.
```

## Plan Before Coding

```text
Before coding, produce:
1. Current behavior.
2. Files likely to change.
3. Risks.
4. Tests to add or update.
5. Small-step implementation plan.

Wait for approval before modifying files.
```
