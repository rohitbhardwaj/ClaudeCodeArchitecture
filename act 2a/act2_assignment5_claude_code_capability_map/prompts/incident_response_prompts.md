# Incident Response Prompts

## Incident Evidence Analysis

```text
Analyze this incident evidence.

Do not modify files.

Review:
1. Recent diffs.
2. Failing tests.
3. Logs.
4. Metrics.
5. Error messages.
6. Likely root causes.
7. Evidence for each hypothesis.
8. Safe next diagnostic steps.

Do not propose a hotfix without listing risks.
```

## Post-Incident Review Draft

```text
Draft a post-incident review.

Include:
1. What happened.
2. Customer impact.
3. Technical impact.
4. Timeline.
5. Root cause hypotheses.
6. Contributing factors.
7. Missing quality gates.
8. Action items.

Mark unknowns clearly.
```
