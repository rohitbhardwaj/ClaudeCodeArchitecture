# Debrief Script

Use this after the classification exercise.

## Opening

Most teams begin with Claude Code as a code helper.

That is natural.

They ask:

```text
Write this function.
Fix this bug.
Add tests.
Clean this file.
```

## Turn

But once Claude Code understands the repository, the question changes.

It is no longer just:

```text
Can it write code?
```

It becomes:

```text
Can it understand context?
Can it preserve behavior?
Can it participate in review?
Can it influence workflow?
```

## Key Point

Level 1 tasks need code review.

Level 2 tasks need architecture context.

Level 3 tasks need engineering governance.

## Strong Close

The more repo-aware Claude becomes, the more architecture-aware your controls must become.

That means:

```text
CLAUDE.md
tests
CODEOWNERS
quality gates
PR templates
security rules
human approval boundaries
```

## Final Line

Claude Code is not only a faster keyboard.

It is a participant in the engineering system.
So architects must design the system it participates in.
