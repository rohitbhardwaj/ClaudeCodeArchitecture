# Assignment Instructions

## Scenario

Your team is beginning to use Claude Code in real repositories.

Some usage is safe and useful.

Some usage may already be drifting toward vibe coding.

This assignment helps you identify the difference.

## Individual Reflection

Answer the following questions:

```text
In my current team:

1. Where are we already doing vibe coding?
2. Which tasks are safe for Claude Code today?
3. Which tasks require architect approval?
4. Which repeated review comment should become written guidance?
5. What is one rule we should add to CLAUDE.md?
```

## Pair Discussion

Pair up with another participant.

Each person shares one answer.

Recommended sharing structure:

```text
My team may be doing vibe coding when...
One safe Claude Code use case is...
One task that requires architect approval is...
One rule I want to add to CLAUDE.md is...
```

## Closeout Question

As a group, answer:

```text
What is one architecture rule that currently lives in people’s heads but should live in the repo?
```

## Output

Each participant should leave with:

```text
one risk area
one safe use case
one approval-required use case
one repeated review comment
one CLAUDE.md rule
```
