# Act 1 Assignment 6 — Personal Reflection

## Theme

Architecture governance begins when architects turn implicit judgment into explicit repository guidance.

## Purpose

This assignment helps architects reflect on where their teams are already practicing “vibe coding” and identify one concrete rule that should move from people’s heads into the repo.

## Individual Reflection

Each architect answers:

```text
In my current team:

1. Where are we already doing vibe coding?
2. Which tasks are safe for Claude Code today?
3. Which tasks require architect approval?
4. Which repeated review comment should become written guidance?
5. What is one rule we should add to CLAUDE.md?
```

## Pair Discussion

Each participant shares one answer with a partner.

## Closeout Question

```text
What is one architecture rule that currently lives in people’s heads but should live in the repo?
```
