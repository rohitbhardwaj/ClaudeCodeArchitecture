# Scoring Rubric — Safe / Ask / Never Policy Design

## 5 — Excellent

Participant classifies all tasks correctly and identifies approval owners.

Strong answer includes:

```text
safe read-only analysis
safe/ask draft tests
ask-first refactor
ask-first API changes
ask-first dependency changes
ask-first payment logic
never silently remove failing tests
safe factual docs
ask-first CI/CD
never PII logging
```

## 4 — Strong

Participant classifies most tasks correctly but misses one approval owner.

## 3 — Acceptable

Participant gets broad categories right but underestimates one high-risk task.

## 2 — Weak

Participant allows too many tasks as safe.

## 1 — Poor

Participant does not define meaningful authority boundaries.
