# Facilitator Guide — Safe / Ask / Never Policy Design

## Learning Objective

Participants learn to define explicit Claude Code authority boundaries.

## Setup Script

Say:

```text
Claude Code cannot scale safely if every developer decides authority boundaries differently.
Safe / Ask / Never gives teams a shared operating model.
```

## 4-Minute Live Version

| Segment | Time |
|---|---:|
| Explain Safe / Ask / Never | 45 sec |
| Participants classify tasks | 2 min |
| Show strong sample | 1 min |
| Debrief | 15 sec |

## 12-Minute Workshop Version

| Segment | Time |
|---|---:|
| Explain policy goal | 2 min |
| Individual classification | 4 min |
| Pair compare | 3 min |
| Discuss approval owners | 2 min |
| Debrief | 1 min |

## Facilitation Prompts

Ask:

```text
What can Claude do read-only?
What requires architecture approval?
What requires security approval?
What should never be allowed silently?
What should be encoded in CLAUDE.md?
What should be enforced by CODEOWNERS?
```

## Strong Closing Line

Claude Code needs explicit authority boundaries before it can scale.
