# One-Page Handout — Claude Code Control Plane Map

## Key Lesson

```text
Governance must live where developers and Claude Code can actually see it.
```

## Control Plane

```text
CLAUDE.md — repo-specific Claude rules
docs/architecture — architecture boundaries
CODEOWNERS — approval owners
PR template — Claude disclosure and rollback
OpenAPI + contract tests — API contract protection
Dependency policy + scan — supply-chain control
Security policy + scan — PII/logging control
Runbook — operating and rollback expectations
CI — automated enforcement
```

## Strong Rule

```text
Put controls where work happens.
```
