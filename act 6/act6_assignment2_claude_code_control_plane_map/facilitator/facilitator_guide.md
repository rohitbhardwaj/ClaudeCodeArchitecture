# Facilitator Guide — Claude Code Control Plane Map

## Learning Objective

Participants learn where to place Claude Code governance controls across repository and operating artifacts.

## Setup Script

Say:

```text
Governance does not work if it only lives in a slide deck.
For Claude Code, controls must live where Claude and developers can see them.
Today we map controls to artifacts.
```

## 4-Minute Live Version

| Segment | Time |
|---|---:|
| Explain control plane | 45 sec |
| Participants map controls | 2 min |
| Show strong sample | 1 min |
| Debrief | 15 sec |

## 12-Minute Workshop Version

| Segment | Time |
|---|---:|
| Explain control categories | 2 min |
| Fill worksheet | 4 min |
| Pair compare | 3 min |
| Inspect sample repo | 2 min |
| Debrief | 1 min |

## Facilitation Prompts

Ask:

```text
Where will Claude read the rule?
Where will reviewers see evidence?
Where will approvals be enforced?
Where will behavior be verified?
Where will rollback be documented?
Which controls should be automated?
```

## Strong Closing Line

Governance must live where developers and Claude Code can actually see it.
