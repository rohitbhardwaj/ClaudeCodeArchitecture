# Debrief Script

## Opening

This exercise is about moving governance from idea to operating system.

## Key Message

A strong control plane uses several artifact types:

```text
CLAUDE.md for AI-readable rules
architecture docs for design boundaries
CODEOWNERS for approval ownership
PR templates for evidence
OpenAPI and tests for contract protection
policies for dependency/security rules
runbooks for rollback
CI for enforcement
```

## Final Principle

If Claude cannot see the rule and the pipeline cannot enforce the rule, governance will depend on memory.

## Closing Line

Governance must live where developers and Claude Code can actually see it.
