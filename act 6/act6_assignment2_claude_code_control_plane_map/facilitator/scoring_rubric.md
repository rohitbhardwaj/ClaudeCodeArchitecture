# Scoring Rubric — Claude Code Control Plane Map

## 5 — Excellent

Participant maps every control to a practical repository or operating artifact and explains why.

Strong answer includes:

```text
CLAUDE.md
docs/architecture
CODEOWNERS
PR template
CI
OpenAPI
contract tests
dependency policy
security/PII policy
rollback runbook
```

## 4 — Strong

Participant maps most controls correctly but misses one enforcement mechanism.

## 3 — Acceptable

Participant maps guidance controls but underuses CI or CODEOWNERS.

## 2 — Weak

Participant relies mostly on informal process.

## 1 — Poor

Participant does not place controls where Claude or developers can see them.
