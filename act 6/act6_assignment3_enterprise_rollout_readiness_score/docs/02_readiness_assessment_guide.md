# Enterprise Rollout Readiness Assessment Guide

## Why Readiness Scoring Matters

Claude Code rollout should not be based on excitement, pressure, or anecdotal success.

Production rollout requires evidence.

A readiness score helps leaders decide:

```text
Can we allow governed rollout?
Should we run a restricted pilot?
Should we limit use to read-only tasks?
Are we not ready for production repositories?
```

## Readiness Principle

```text
Score evidence, not intention.
```

## Rating Scale

### 5 — Strong

Control exists, is current, used, and enforced.

### 4 — Good

Control exists and is mostly used, but needs minor improvement.

### 3 — Partial

Control exists but is incomplete, inconsistent, or weakly enforced.

### 2 — Weak

Control exists informally or only in some repos.

### 1 — Missing

Control is absent or not usable.

## Recommended Restrictions by Score

### 40–50

Ready for governed rollout.

### 30–39

Pilot with restrictions.

### 20–29

Limited use only.

### Below 20

Not ready for production repositories.
