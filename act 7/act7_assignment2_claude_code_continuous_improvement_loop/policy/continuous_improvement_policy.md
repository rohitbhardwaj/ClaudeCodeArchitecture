# Claude Code Continuous Improvement Policy

## Purpose

Ensure Claude Code governance improves based on real usage.

## Monthly Review Inputs

```text
Claude-assisted PRs
review comments
failed CI checks
security findings
rollbacks/incidents
developer feedback
reviewer feedback
repeated prompt failures
```

## Required Improvement Actions

At least monthly, teams must consider updates to:

```text
CLAUDE.md
prompt templates
PR template
review checklist
CI gates
security scans
runbooks
training
katas
```

## Key Rule

```text
Claude Code governance is not one-time setup; it is a learning loop.
```
