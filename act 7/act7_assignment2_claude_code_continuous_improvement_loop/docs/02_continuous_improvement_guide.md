# Claude Code Continuous Improvement Loop Guide

## Why a Learning Loop Matters

Claude Code governance is not finished when you create `CLAUDE.md`, a PR template, and a runbook.

Teams learn from real usage.

Monthly review should ask:

```text
What did Claude-assisted work improve?
What failure patterns repeated?
What did reviewers struggle to evaluate?
What CI checks failed?
What security findings appeared?
What incidents or rollbacks happened?
What prompts confused Claude?
What training gaps appeared?
```

## Continuous Improvement Targets

The feedback loop should improve:

```text
CLAUDE.md
prompt templates
PR checklist
CI gates
security policies
test expectations
runbooks
team katas
reviewer training
```

## Strong Rule

```text
Every month, convert repeated failure patterns into better repo rules, prompts, tests, checks, or training.
```

## Monthly Loop

```text
1. Collect evidence.
2. Identify repeated patterns.
3. Decide improvement actions.
4. Update control artifacts.
5. Train the team.
6. Measure next month.
```
