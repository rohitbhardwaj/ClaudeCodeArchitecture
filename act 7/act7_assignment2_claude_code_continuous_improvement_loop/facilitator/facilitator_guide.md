# Facilitator Guide — Claude Code Continuous Improvement Loop

## Learning Objective

Participants learn to design a monthly feedback loop that improves Claude Code governance over time.

## Setup Script

Say:

```text
Governance is not done after you create the first CLAUDE.md.
The team must learn from real Claude-assisted work and improve the controls every month.
```

## 5-Minute Live Version

| Segment | Time |
|---|---:|
| Explain learning loop | 1 min |
| Participants fill worksheet | 2 min |
| Show strong sample | 1 min |
| Debrief | 1 min |

## 15-Minute Workshop Version

| Segment | Time |
|---|---:|
| Explain monthly evidence sources | 2 min |
| Review sample evidence | 4 min |
| Fill worksheet | 4 min |
| Build improvement backlog | 3 min |
| Debrief | 2 min |

## Facilitation Prompts

Ask:

```text
What failure patterns repeated?
What should be added to CLAUDE.md?
What prompt template is missing?
What should PRs disclose?
What CI gate would catch the issue?
What training or kata should be added?
```

## Strong Closing Line

Claude Code governance is not one-time setup; it is a learning loop.
