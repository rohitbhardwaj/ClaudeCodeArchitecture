# Scoring Rubric — Claude Code Continuous Improvement Loop

## 5 — Excellent

Participant defines a full monthly loop with evidence sources, artifact updates, owners, and learning summary.

Strong answer includes:

```text
PR review
review comments
CI failures
security findings
rollbacks/incidents
developer feedback
reviewer feedback
prompt failures
artifact updates
new kata
monthly summary
```

## 4 — Strong

Participant includes most inputs and improvement actions.

## 3 — Acceptable

Participant includes review process but lacks concrete artifact updates.

## 2 — Weak

Participant collects feedback but does not turn it into controls.

## 1 — Poor

Participant treats governance as one-time setup.
